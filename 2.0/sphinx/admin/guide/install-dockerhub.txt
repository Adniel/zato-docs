Zato from DockerHub
===================

.. image:: /gfx/admin/guide/dockerhub.png
   :width: 100%

Requirements
------------

* Docker 1.0.1+

Usage
-----

Quickstart cluster
~~~~~~~~~~~~~~~~~~

* Get Docker image with a preinstalled Zato from `DockerHub <https://hub.docker.com/r/zatosource/zato-2.0.7-quickstart/>`_.

::

    host$ docker pull zatosource/zato-2.0.7-quickstart

* Create Zato quickstart environment and generate passwords for zato user and web-admin:

::

    host$ docker run --name="zato-quickstart" zatosource/zato-2.0.7-quickstart /bin/bash \
        -c 'quickstart_create.sh'

* Commit the changes to the image to make them persisent in subsquently created containers:

::

    host$ docker commit -m "<comment>" \
        -a "<username>" zato-quickstart zatosource/zato-2.0.7-quickstart:<tag>

For example:

::

    host$ docker commit \
        -m "Created a Zato quickstart environment and generated new passwords" \
        -a "myuser" zato-quickstart zatosource/zato-2.0.7-quickstart:myimage

* Retrieve your web admin password. The password will be printed out on terminal:

::

    host$ docker run zatosource/zato-2.0.7-quickstart:myimage /bin/bash \
        -c 'cat /opt/zato/web_admin_password /opt/zato/zato_user_password'

* Create a container in which Zato components will be launched:

::

    host$ docker run -it -p 22 -p 6379:6379 -p 8183:8183 -p 17010:17010 -p 17011:17011 \
        -p 11223:11223 zatosource/zato-2.0.7-quickstart:myimage

That concludes the process - a web-admin instance is running on http://localhost:8183
and you can log into it with the username 'admin' using the password printed
on the terminal above.

You can also connect via SSH to the container
under which Zato is running. User: zato. Password: second one of the two printed
on terminal above.

:doc:`documentation index <../../index>`
now.
